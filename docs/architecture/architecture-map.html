Research Copilot — Architecture Map

AI research-assistant desktop app. Electron shell over pi-mono agent runtime, React + Zustand UI, and a framework-independent lib/ brain (agents, tools, skills, storage, compute, telemetry). This page is the human navigator; the sibling architecture-map.json is the machine spec for the feature-building agent.
642 tracked files lib/ 411 app/ 126 ipc.ts 3029 lines coordinator.ts 1403 stack: Electron · React · Zustand · pi-mono · OTel
"The system does not pursue complex architecture to guarantee quality. Instead, it pursues minimum discipline to guarantee survival + evidence-driven incremental improvement."

01 Overview & Stack

Three layers: the Electron app/ shell, the reusable shared-* packages, and the lib/ agent brain. The brain is framework-independent and holds the bulk of the logic.

app/ 126

Electron desktop: src/main (lifecycle + ~88 IPC + agent wiring), src/preload (context bridge), src/renderer (React UI + 27 Zustand stores).

lib/ 411

Framework-independent brain: agents, tools, skills, memory-v2, compute, telemetry, wiki, reports, sharing, importers. The part that thinks.

shared-ui/ 26

Reused React/Zustand: tool-renderer registry, ModelSelector, ReasoningToggle, activity/progress/usage stores, model constants.

shared-electron/ 11

Reusable IPC utils: api-key-loader (config>env priority), file-tree, ipc-base, jsonl, recent-projects.

Agent runtime pi-mono

@mariozechner/pi-coding-agent + pi-agent-core + pi-ai. Provides the Agent loop, coding tools (read/write/edit/bash/grep/find), and LLM integration.

Build & Test

npm run dev = electron-vite dev (3 targets). npm test = custom runner (scripts/run-tests.mjs) with suites: memory-v2, agents, telemetry, ledger, renderer, …

02 Process Model & Data Flow

Classic Electron 3-process split. Every renderer↔backend call crosses the preload bridge; the agent turn fans callbacks back as broadcasts.

▸ Renderer
app/src/renderer/ — React + Zustand
  • App.tsx → Left | Center | Right | StatusBar
  • 27 stores subscribe to window.api.on*
  • session.hasProject gates the whole UI
▸ Preload Bridge
app/src/preload/index.ts (722)
  • invoke() request/response
  • subscribe() → disposer (events)
  • send() fire-and-forget
  • exposes window.api
▸ Main Process
app/src/main/{index,ipc}.ts
  • ~88 IPC handlers (ipc.ts)
  • ensureCoordinator → createCoordinator
  • per-window state + telemetry tracer
  • calls into lib/
▸ round-trip: Renderer window.api.x() → preload invoke → ipcMain.handle (ipc.ts) → lib/ command → return → Zustand → re-render ▸ agent turn: sendMessage → 'agent:send' → ensureCoordinator (recreate if model/auth changed) → createCoordinator(onStream, onToolCall, onToolResult, onUsage, tracer…) ▸ agent turn: coordinator: detect intent (rules) + match skills (LLM) → assemble tools → build system prompt → pi-mono Agent runs (transient-retry wraps) ▸ agent turn: hooks fire callbacks → safeSend('agent:stream-chunk' | 'agent:activity' | 'agent:usage' …) → stores → 'agent:done'

03 Subsystem Map

Each lib/ subsystem is self-contained. Find the one your feature touches, then jump to its extension recipe.

agents lib/agents/

Coordinator orchestrator (1403 lines): creates the pi-mono Agent, detects intent, matches skills (LLM, ≤5), context compaction, telemetry adapter, transient-retry. Prompt registry in prompts/index.ts.

tools lib/tools/

createResearchTools(ctx) assembles native pi-mono AgentTools: web, literature, fulltext, convert, data-analyze, diagram, artifacts, memory, wiki, compute, S3. Categories drive telemetry stamping.

memory-v2 lib/memory-v2/

RFC-014 files-as-carrier artifact store. Notes=.md, papers=.bib+sidecar, data=.rp.yaml under rp-artifacts/; .research-pilot/index/ is a derived cache. Naming: <title-slug>-<id8>.

memory lib/memory/

Auto-Memory: .research-pilot/memory/<type>/<slug>.md + a Memory Index in agent.md. Types user|feedback|project|reference. Distinct from memory-v2; both current.

commands lib/commands/

Thin command layer over the store: artifact CRUD, list/search, upsertPaperArtifact (dedup, fill-only merge), enrichment, session-summary, memory-explain.

mentions lib/mentions/

@(note|paper|data|file|url):key parse → resolve (id→prefix→citeKey→title) → candidates (entity-index + git ls-files). Cache invalidated on artifact CRUD.

skills lib/skills/

SKILL.md system. loader.ts merges builtin + ~/.research-pilot + workspace (later wins). 19 builtin skills. Loaded on demand via the load_skill tool.

compute lib/compute/ +3

RFC-008 ComputeBackend interface + Registry + PlanStore. Local / Modal / AWS-EC2 instances. Plan→approve→execute gate. Adding a backend ≈ 400–800 LOC (§19 checklist).

telemetry lib/telemetry/ (48)

OTel PipilotTracer → JSONL spans. turn.id auto-propagates via OTel context (TURN_ID_KEY). Opt-in (default off), per-collaborator. Blob store for >4KB content.

ledger lib/ledger/ (6)

Append-only logs: artifact ledger, memory ledger, user-response-signals, view-log. Writers auto-fill traceId/spanId/turnId from active context.

audit-graph lib/audit-graph/ (6)

Trust & Audit v0.8. Pure provenance graph from spans+ledgers+index → Audit tab. Citation resolvability (A1) flags suspicious/fabricated citations.

wiki lib/wiki/ (20)

RFC-005 background agent builds a paper/concept library with memory sidecars; backs wiki_* tools. Pause/resume around the coordinator. Epistemic contract: never cite raw memory values.

reports lib/reports/ (16)

RFC-007 Paper Pack — onboarding reading path. input→aggregate→rank→synthesize(LLM)→render md+html. Hash-cached.

sharing lib/sharing/ (9)

RFC-013 git-backed PI→student collab. One private repo, single main, lane=folder. share/sync/poll/invite/accept + AI-merge conflicts. Detect-but-never-auto-apply.

importers / fulltext lib/importers/ · fulltext/

BibTeX bulk import (dedup, fill-only merge, RFC-006). resolveFulltext(): cache → Paperclip MCP → arXiv direct.

07 Compute Subsystem RFC-008

One ComputeBackend interface + Registry + PlanStore. Local / Modal / AWS-EC2 are instances. The plan→approve→execute gate keeps cost-bearing work behind explicit user approval.

ComputeBackend lib/compute/backend.ts

  • probeAvailability() · plan(input) · submit(plan,opts)
  • getStatus · waitForCompletion · stop · destroy
  • hydrate() → {run,status} (crash recovery, A3)
  • Identity {id, displayName, toolPrefix} (A4) + Capabilities {requiresApproval, hasCost, supportsGpu, supportsStop, supportsStreaming}

Registry & PlanStore

  • register() validates id/toolPrefix uniqueness
  • plan() captures immutable effectiveRequiresApproval (A1)
  • approve/reject/discardPlan, submit() (in-flight memo)
  • PlanStore = persistent ~/.research-pilot/compute-plans.json

Backends

  • local — no approval/cost, in-proc or Docker, preflight/smoke/full phasing
  • modal — approval + cost + GPU; plan-agent inspects script; backend-owned cost-kill (A2)
  • aws-ec2 — approval + cost + GPU; Phase 1 user supplies InstanceSpec; provisions EC2 + SSH

States & flow

plan: pending → approved | rejected | discarded

run: pending_approval | queued | running | completed | failed | timed_out | cancelled | cost_killed

compute_plan → persist (gate if requiresApproval) → user approves (IPC) → <prefix>_execute → backend emits run-update/complete via ctx.emit → compute:event → store.

AWS credential chain lib/aws/credentials.ts

per-call opts → Settings UI → env (AWS_ACCESS_KEY_ID / AWS_REGION / AWS_DEFAULT_REGION) → ~/.aws/credentials profile → EC2 instance metadata

Region & keys resolve independently. Matches the global config-priority rule — inverting it lets stale shell exports shadow fresh UI keys.

08 Telemetry & Audit

OpenTelemetry traces every turn / step / tool / LLM call into JSONL spans; ledgers record artifact & memory events; the audit-graph derives a provenance graph for the Audit tab. Opt-in (default off), per-collaborator.

Tracing lib/telemetry/ (48)

  • PipilotTracer — one TracerProvider/main; AsyncLocalStorage context; startSpan() auto-stamps project/session/turn.id
  • TraceStore — ring queue → .research-pilot/traces/spans.*.jsonl (OTLP/JSON)
  • TURN_ID_KEY propagates turn.id through OTel context (recent work)
  • blob-store for >4KB content; redaction scrubs PII
  • dev guard throws on unknown pipilot.* attribute keys

Ledgers lib/ledger/ (6)

  • artifact ledger → artifacts/ledger.jsonl (create|edit|delete|convert|export|…)
  • memory ledger → memory-v2/ledger.jsonl
  • user-response-signals · view-log
  • writers auto-fill traceId/spanId/turnId from active context

Audit-graph lib/audit-graph/ (6)

  • Nodes: session|trace|step|tool|chat|artifact|file|dir|span
  • Edges: contains|precedes|invokes|reads|writes|creates|retrieved|mentions…
  • projectGraph() pure materializer → audit:get-graph → Audit tab
  • Citation resolvability (A1) flags suspicious / fabricated citations

What to emit when adding a tool

  • Register name→category in lib/tools/categories.ts
  • telemetry-adapter auto-creates execute_tool {name} spans
  • On error: span.setAttribute('pipilot.tool.error_class', …)
  • If it writes artifacts: append an artifact-ledger row (turnId auto-fills)

10 Extension Recipes

The canonical "where do I add code" answers. The JSON spec has these too, with file anchors — feed it to the feature agent.

Add a research tool

lib/tools/index.ts:33–95
  1. Create lib/tools/<name>.ts exporting createXTool(ctx): AgentTool (Type.Object params, execute() → toAgentResult()).
  2. tools.push(createXTool(ctx)) in index.ts (gate behind a ctx flag if it needs optional creds — see compute/s3).
  3. Add name→category in lib/tools/categories.ts (telemetry).
  4. Add a render config in shared-ui/tool-renderers/registry.ts (icon, summaries).
  5. If it writes artifacts, append an artifact-ledger row.

Add an IPC channel (end-to-end)

app/src/main/ipc.ts → preload → renderer
  1. main: ipcMain.handle('domain:verb', …) (or the per-window wrapper); return {success, data?|error}.
  2. preload: add the method to ElectronAPI + api via invoke()/subscribe()/send().
  3. renderer: call window.api.<method>().
  4. For pushes: safeSend(win,'event',payload) + a subscribe() bridge.

Add a compute backend

~400–800 LOC · docs/spec/compute.md §19
  1. Create lib/compute/backends/<id>/<id>-backend.ts implementing ComputeBackend.
  2. Declare Identity (toolPrefix /^[a-z][a-z0-9_]*$/) + Capabilities; define JSON-only PlanData/RunData + version (A5).
  3. Implement all methods; emit run-update/complete via ctx.emit; own the cost-kill if hasCost (A2).
  4. Register in lib/agents/coordinator.ts: computeRegistry.register(new XBackend(ctx, deps)).
  5. Surface settings in ComputeSettings.tsx; add a compute-<id> skill + tests.

No new IPC / preload / store wiring — compute:event + onComputeEvent + applyEvent carry it.

Add a skill

  1. Create lib/skills/builtin/<name>/SKILL.md (front-matter + body).
  2. loader.ts auto-discovers — no registration.
  3. Or drop into workspace/.research-pilot/skills/ for project-only.

Add a prompt

  1. Add a key→string entry in lib/agents/prompts/index.ts.
  2. Use via loadPrompt('key'). Keep strings inline (bundler-safe).

Add a store / UI tab

  1. Create stores/<x>-store.ts; subscribe to window.api.on* at App mount.
  2. Extend the LeftTab/CenterView union in ui-store.ts; branch in the panel.
  3. Reset in session-store; persist global prefs in both init + reset().

Add an artifact type

  1. Extend ArtifactType + define <X>Artifact in lib/types.ts.
  2. serialize/parse in artifact-files.ts; path in artifact-writer.ts; scan in indexer.ts.
  3. Update commands/{list,search}.ts + UI tabs.

11 Conventions & Gotchas

Hard-won rules. Violating these has cost releases. The feature agent must respect them.

Reply prefix

Always reply starting with "Hey, Captain!" (CLAUDE.md).

Config priority: UI > env

explicit per-call > Settings UI > shell env > default. Inverting lets stale shell-exported keys shadow fresh UI keys silently. Guarded by api-key-priority.test.ts. Dev/debug flags (RESEARCH_COPILOT_DEBUG, ENABLE_COMPUTE_STUB) and AWS_*_FILE paths are intentionally env-only.

CJS imports in main process

In app/src/main/, default-import CJS pkgs (electron-updater, node-pty): import x from 'pkg'; const {y}=x. Named imports break the packaged ESM build — cost a release.

Persisted UI state in reset()

localStorage-backed prefs (leftSidebarWidth/collapsed, markdownEditMode) must be re-read in both the store's initial value and reset() — session-store calls ui-store.reset() on every project open/close.

Workspace folder = .research-pilot only

.agentfoundry/ is NOT created by runtime code (only in ignore lists / RFC-001). No task-anchor/focus mechanism — planning is model-maintained; cross-session state = session-summaries + recaps + artifacts + memory.

'dim' is a CSS layer, not a theme

Dim = html.dark.dim, not a 3rd value in the theme union. Don't collapse it.

Telemetry is opt-in & per-collaborator

Default off; toggle in gitignored .research-pilot/preferences.json. RFC-013 §8 prevents a PI from flipping everyone's tracing on.

Global keydown guard

Renderer global keydown handlers on plain keys must bail when a modal is open or focus is editable (a ⌫ handler once deleted recents while typing in a modal).

Stacked-PR --delete-branch cascade

Merging A in A→B→C with --delete-branch auto-closes B/C and they can't reopen. Retarget downstream PRs first or collapse the stack.

main branch protection

required build check, strict mode, no force-push/delete; admins bypass (enforce_admins:false) so direct version-bump pushes work.
