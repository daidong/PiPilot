# YOLO-Researcher v2.2: Single-Root Path Contract (RFC-007)

> Goal: remove path chaos by defining one global, canonical filesystem layout.
> One workspace = one session root. No projectId namespace. No dual artifact roots.

---

## 0. Decision (Non-Negotiable)

All runtime outputs and all evidence pointers use a single root:

- `runs/turn-xxxx/...` (resolved under `<workspace>/`)

The following are **not** runtime evidence roots:

- `.yolo-researcher/library/...`
- `workspace/runs/turn-*` (literal `workspace/` prefix inside path strings)
- `yolo/<projectId>/runs/...`

If a tool/skill returns files outside `runs/turn-xxxx/...`, they are not canonical evidence.

---

## 1. Canonical Workspace Layout

```text
<workspace>/
  PROJECT.md
  FAILURES.md
  user-input-queue.json
  runs/
    turn-0001/
      action.md
      result.json
      cmd.txt
      stdout.txt
      stderr.txt
      exit_code.txt
      artifacts/
        tool-events.jsonl
        agent-output.txt
        <all tool/skill outputs for this turn>
    turn-0002/
      ...
  .agentfoundry/
    skills/
      ...
```

Rules:

1. `runs/` is the only turn-history root.
2. `turn-xxxx/` is append-only.
3. `artifacts/` under each turn is the only output location for tool/skill generated files.

---

## 2. Canonical Path Constants

For turn `N`:

- `TURN_ID = turn-${String(N).padStart(4, '0')}`
- `TURN_DIR = runs/${TURN_ID}`
- `ARTIFACTS_DIR = runs/${TURN_ID}/artifacts`

Every runtime component (session, llm-agent, wrappers, skills bridge, UI references) must derive paths from this contract only.

---

## 3. Evidence Pointer Contract

All `PROJECT.md` evidence pointers use workspace-relative canonical paths:

- `facts[].evidencePath`
- `constraints[].evidencePath`
- `done[].evidencePath`
- `claims[].evidencePaths[]`
- `planBoard[].evidencePaths[]`
- `keyArtifacts[]`

Canonical format:

- `runs/turn-xxxx/...`

No other prefix is valid as canonical evidence reference.

---

## 4. Tool/Skill Output Contract

All wrappers and skill calls must write outputs under current turn artifacts:

- Literature: `runs/turn-xxxx/artifacts/literature/...`
- Markitdown/conversion: `runs/turn-xxxx/artifacts/converted/...`
- Data analysis: `runs/turn-xxxx/artifacts/data-analysis/...`
- Writing: `runs/turn-xxxx/artifacts/writing/...`

Wrapper default `outputDir` behavior:

1. If caller provides `outputDir`, it must be relative and inside current `ARTIFACTS_DIR`.
2. If caller omits `outputDir`, wrapper auto-uses its category subdir under current `ARTIFACTS_DIR`.
3. Wrapper must return canonical relative paths (`runs/turn-xxxx/...`) in structured results.

---

## 5. LLM Prompt Contract

Prompt examples and schema examples must always show canonical paths only:

- `runs/turn-xxxx/...`

Prompt text must not suggest any alternative output root (such as `.yolo-researcher/library/...`).

---

## 6. No Compatibility Layer

This RFC intentionally does **not** add migration/legacy compatibility for old path formats.

Policy:

1. New runs follow this contract only.
2. Old non-canonical paths are ignored for new evidence writing.
3. Code paths producing old formats are bugs and must be removed.

---

## 7. Acceptance Criteria

A build is compliant only if all pass:

1. No prompt string in yolo v2 references `.yolo-researcher/library` as outputDir.
2. Wrappers default output dirs resolve under `runs/turn-xxxx/artifacts/...`.
3. `result.json`, `action.md`, and `PROJECT.md` pointers use `runs/turn-xxxx/...` only.
4. End-to-end literature turn succeeds without any `PROJECT.md update rejected: ... evidence path ...` error caused by mixed roots.

---

## 8. Scope Notes

This RFC is about path coherence only.

Out of scope:

- quality/stagnation policy tuning
- stage progression semantics
- review strategy changes
