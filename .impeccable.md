# AgentFoundry — Impeccable Design Context

> Single source of truth for design decisions in this codebase. Every
> impeccable skill reads this file before producing output. Update it
> when direction shifts — stale context produces generic work.

## Design Context

### Users

- **Primary**: working researchers, PhD students, academic authors
  deep in literature review, paper drafting, or data exploration.
- **Context of use**: hours-long focus sessions on macOS desktop.
  Often paired with a terminal, a PDF reader, and a notes app.
  Rarely under time pressure, always under cognitive load.
- **Job to be done**: externalize a research workflow into an AI-
  assisted surface — literature search, artifact capture, data
  analysis, cumulative wiki memory — without losing the rigor,
  density, and control of a real desktop tool.
- **Assumptions**: keyboard-fluent, comfortable with shortcuts and
  command palettes, aesthetically literate (they read design blogs
  too), allergic to marketing-ware.

### Brand Personality

Three words: **Precise · Calm · Deep**.

- **Voice**: direct, technical, unfussy. Labels are nouns, not
  slogans. Errors name the cause, not the vibe. No emoji-prose, no
  exclamation points, no "magic" language.
- **Emotional goal**: *confident command.* The moment a user opens
  the app they should feel "this is a real tool" — in the lineage of
  Linear, Raycast, Arc DevTools — not a demo, not a toy, not an ad.
- **What the user should NOT feel**: overstimulated, rushed, sold to,
  or talked down to.

### Aesthetic Direction

**References**
- **Arc Browser / Raycast** — quiet surfaces, spring-physics micro-
  motion, refined typographic rhythm, confident use of a single
  accent. Craft-in-the-small. Design that disappears into usability.
- **Linear** — density-first, keyboard-first, status chips and
  command palettes that feel like second nature.
- **iA Writer / Craft (for reading states)** — calm focus, generous
  vertical rhythm in prose surfaces only.

**Anti-references** (all four explicitly called out by the user)
- ❌ **Decorative motion.** No bouncing gradients, particle effects,
  scroll-linked parallax, shimmer, or hero loops. Every animation
  must earn its place by encoding state change, spatial continuity,
  or progress.
- ❌ **Consumer-SaaS "rounded-2xl + huge whitespace."** This is not
  a pricing page. Info density is an academic virtue. We reject
  airiness for its own sake.
- ❌ **Heavy-handed AI-ness.** No sparkle icons, no purple glow, no
  "✨ Ask the AI" labels, no robot avatars, no "magic" framing. AI
  features should feel embedded and trustworthy, not spectacular.
- ❌ **Traditional academic software drabness.** No Zotero-grey,
  no LaTeX-editor chrome, no menus-everywhere, no "we don't care
  about design." We target users who read academic papers AND read
  design blogs.

**Theme**
- **Dark-first.** `html.dark` is the primary experience; light mode
  is secondary and tuned for daylight workflows, not marketing
  screenshots. The warm navy-tinted neutrals (`#141a1e` → `#232b32`)
  are the signature and must not drift toward cold grey.
- **Primary accent**: teal (`#34d0ba` dark / `#0d9488` light).
- **Secondary accent**: indigo (`#818cf8` dark / `#6366f1` light).
- **Gradient motif**: the teal → indigo horizontal gradient
  (`.t-gradient-accent-h`) is the closest thing to a logo and is
  reserved for deliberate moments (progress fills, active focus
  markers, the hero-ish state of the left sidebar tool). Use rarely;
  it loses meaning if it's everywhere.

**Typography**
- **Inter variable**, with `cv01-04` feature settings enabled. The
  type scale (see `global.css` comments, 1.2 ratio / Minor Third) is
  authoritative: 10/11/13/14/17/20/24 px. Do not introduce new sizes.
- Tabular numerals (`tabular-nums`) on everything numeric — scores,
  counts, timestamps, progress.
- Uppercase small-caps section labels (tracking-wider, `text-[10px]`)
  are part of the visual grammar; use for section headers, never for
  body copy.
- Monospace (`SF Mono / Monaco / Menlo / Fira Code`) for code, IDs,
  and paths, with a teal-tinted surface treatment.

### Design Principles

1. **Density is a feature, not a bug.** Never trade information
   density for whitespace unless the tradeoff is measured and the
   reader benefits. Arrange, group, and align — don't dilute.

2. **Every motion encodes state.** Spring physics, 150–300ms
   easings, small displacements. No motion without a reason: loading
   → pulse; progress → width; transitions → spatial continuity.
   When in doubt, cut the animation.

3. **One accent, used sparingly.** Teal primary, indigo secondary,
   gradient as an event. Status colors (success/error/warning/info)
   are their own vocabulary and must never be decorative. Muted
   neutrals do 80% of the work.

4. **Keyboard is a first-class surface.** Visible focus rings, skip
   links, command palette, `kbd` chips for shortcuts. A cursor user
   and a keyboard user should reach the same place in the same
   number of cognitive steps.

5. **AI presence is quiet.** Agent status shows as a pulse dot, not
   a sparkle. Wiki builds in the background, not on a splash screen.
   Chat is a panel, not the product. The product is the research,
   not the AI.

6. **Craft lives in the small.** A 2px accent bar, a 1px border
   alignment, a 150ms transition, a tabular number that doesn't
   reflow — these are where the product earns the user's trust.
   Audit every corner before touching the big layout.

7. **Respect the user's environment.** Honor `prefers-reduced-
   motion`, `prefers-color-scheme`, OS text scale. Don't assume
   broadband, don't assume a trackpad, don't assume English.
